The wbc database enables sharing information across different (independently executable) code components:

1. SParserData : Contains all the static data
    i) SWorld : The world (physics) specification

   ii) CMappedList<SGraphics> : A string-indexed linked-list of graphics view specifications
       A) SGraphics : A camera and one or more lights (defines a graphics view of the world)

  iii) CMappedList<SRobot> : A string-indexed linked-list of robot specifications
       A) SRobot : Contains a branching structure (string-indexed tree) of robot links
          a) SRobotLink : Contains the inertial and configuration (wrt a parent) 
                          properties of a link.

2. SControllerData : Contains dynamic data generated by the controller
    i) CMappedList<STaskController> : A string-indexed linked-list of controllers 
                                   for one or more robots
       A) STaskController : A controller for one behavior to be executed on a robot
          a) SRobotSensorData : Joint angles, sensor info
          b) SRobotActuatorData : Command torques/forces
          c) SRobot : A const pointer to the static robot data (which is on the pile).
          d) SServo : Data for op-space to gen-coords force transformations and 
                      null-space projections
          e) SJSpaceModel : The joint space model
          f) CMappedList<STask> : A string-indexed linked-list of operational space models 
                               for a set of executing tasks

3. SGuiData : Stores all the data generated by the gui as it runs
    i) CMappedList<SChaiGraphics> : A string-indexed linked-list of currently 
                                 executing chai instances
       A) SChaiGraphics : 
          a) cWorld : Chai3D world
          b) cCamera : Chai3D camera
          c) CMappedList<SGraphicsPhysicalLink> : A string-indexed linked-list for all
                                               the "inertial" objects to be rendered.
             I) SGraphicsPhysicalLink : Contains SRobotLink (robot definition) and
                                        CGenericObject (chai obj) pointers. Obviates 
                                        dupe name-link and name-chai-object lookups.
          d) CMappedList<SGraphicsMesh> : A string-indexed linked-list for all the "purely
                                       graphical (zero mass)" objects to be rendered.
             I) SGraphicsMesh : Translation and rotation information for the object 
                                from the base rendering frame

4. SIOData : Stores all the sensor data obtained from the simulation/real world
    i) CMappedList<SRobotIOData> : A string-indexed linked-list of robot I/O 
                                modules (sensors and actuators)

5. Dynamic typing : Add your type to DbDynamicTypeSpecs.hpp/cpp (look at the existing examples).
   NOTE : Controller types MUST be dynamically typed, and must be registered with SDatabase::types_
